# Forecasting Automotive Radar Interference: Is Current Spectrum Enough?

Alessandro Bazzi<sup>#\$</sup>, Francesco Miccoli<sup>\$</sup>, Zhuofei Wu<sup>\$\*</sup>, Fabrizio Cuccoli<sup>+</sup>, Vincent Martinez<sup>§</sup>

<sup>#</sup>DEI, Università di Bologna, Italy

<sup>\$</sup>National Laboratory of Wireless Communications (WiLab), CNIT, Italy

College of Computer Science and Technology, Harbin Engineering University, China

<sup>+</sup>National Laboratory of Radar and Surveillance Systems (RaSS), CNIT, Italy

<sup>§</sup>NXP Semiconductors, France

Abstract — Vehicles are increasingly equipped with advanced driver-assistance systems (ADAS) sensors that help reduce road risks and accidents. Among them, radar sensors play a key role due to their affordable cost and high effectiveness in object detection and tracking. However, with increasing penetration, radar signals generated by different vehicles can create interference and this may have a detrimental impact on the functionality. This observation is leading to a discussion about the need for new bands and mitigation rules. Assuming frequency modulated continuous waveform (FMCW) and a realistic distribution of vehicles on a highway, in this work, we: (i) illustrate a model for the investigation of the interference impact; (ii) propose to use a chirp-by-chirp frequency hopping to mitigate the interference and compare it with the frame-by-frame frequency hopping; (iii) show results, highlighting that both mitigation mechanisms and large frequency bandwidths are necessary in the long run.

Keywords — Automotive radar, FMCW, interference mitigation.

## I. INTRODUCTION

In the automotive world, the number of radars fitted to cars is expected to grow more and more, with the aim of increasing the level of driver assistance systems until, in the future, autonomous driving is achieved. One of the main problems caused by the increasing penetration rate of radar systems is interference from surrounding vehicles, which leads to performance degradation [1].

To counteract the issue of interference and host new applications, new frequency bands are being considered and new techniques are being studied to mitigate the interference phenomenon. Regarding the new spectrum, one possible direction currently under discussion is the exploitation of new bands around 140 GHz [1], [2]. Regarding interference mitigation, several mechanisms have been described and tested in prior studies, with relevant examples in the MOSARIM and IMIKO Projects [3].

The present study focuses on the most common modulation for automotive radars, frequency modulated continuous waveform (FMCW), and on mitigation proposals that do not require coordination between devices. This includes randomization of the starting frequency at the beginning of a sequence of chirps (also called frame), hereafter called frame-by-frame frequency hopping. However, this approach may be not sufficient in the long term, due to the increasing penetration of radar devices and the limited available spectrum.

In this work, we therefore propose to randomly modify the starting frequency at the beginning of each chirp, which we call chirp-by-chirp frequency hopping, and we evaluate the improvement through a novel mathematical model. As an output, we derive the probability that a number of subsequent frames are lost as a function of various parameters, such as the signal bandwidth and the penetration rate in realistic highway scenarios.

## II. REALISTIC TRAFFIC SOURCE

The provided interference evaluation is based on realistic positions of the vehicles as derived from the open source simulator WiLabVIsim described in [4].<sup>1</sup> The simulator obtains the position of the vehicles from the traffic simulator SUMO [5] and identifies the presence of potential interferers, calculating all the characteristics of the paths taken by the interfering signals. The output of WiLabVIsim is independent of the specific radar signal settings, such as waveform, bandwidth, signal processing algorithms used, or methods to reduce the effect of interference [4].

## III. INTERFERENCE MODELING

In this section, we derive mathematical models to quantify the impact of interference, based on the output of the WiLabVIsim simulator. The model evaluates how frequently a radar system is subject to issues due to interference, assuming the use of FMCW and all radars with similar settings in terms of bandwidth and slope.

The starting assumptions are:

1) The available bandwidth is $B _ { \mathrm { T O T } } ;$

2) The chirp bandwidth is $B _ { \mathrm { c h } } ;$

3) The analog to digital conversion bandwidth is $B _ { \mathrm { A D C } } ;$

4) Each chirp is sent within a repetition time interval denoted as $T _ { \mathrm { r c h } }$ , which includes the chirp transmitting time $T _ { \mathrm { c h } } ;$ the starting instant of $T _ { \mathrm { c h } }$ inside $T _ { \mathrm { r c h } }$ varies from chirp to chirp to introduce some variability; the empty interval also allows to reset the oscillators;

5) Each radar sends a train of chirps, called frame, with an empty interval in the end; the number of chirps in a frame is denoted as $N _ { \mathrm { c h } }$ and the time portion in active state compared to the sum of active and inactive states is called duty cycle and denoted as $\delta ;$ the duration of a frame, including active and inactive states, is denoted as $T _ { \mathrm { r f } }$ and can be calculated as

$$
T _ { \mathrm { r f } } = T _ { \mathrm { r c h } } \cdot N _ { \mathrm { c h } } / \delta \ ;\tag{1}
$$

6) We assume unslotted use of frequency and time domains, which means that there are no restrictions about when a chirp starts in time and frequency, apart from the constraints related to the total available bandwidth and the chirp and frame duration;

7) There is one radar for which performance is studied, hereafter called victim, and $N _ { \mathrm { i n t } }$ interfering radars, called attackers; the terminology used is for clarity and does not mean that there is deliberate intention to reduce the effectiveness of victim radar sensing;

8) One chirp of the victim is considered lost if there is another signal sent by an attacker including a chirp that overlaps with that of the victim by at least $x _ { \mathrm { f } } \in [ 0 , 1 ]$ in the frequency domain and $x _ { \mathrm { t ( C ) } } \in [ 0 , 1 ]$ in the time domain;

9) One frame of the victim is considered lost if $K _ { \mathrm { c h } }$ or more chirps are lost;

10) The radar system yields an application-level failure if $M _ { \mathrm { f } }$ or more consecutive frames are lost.

To mitigate mutual interference without coordination between radar sensors, two approaches are compared:

• Frame-by-frame frequency hopping: the starting frequency changes randomly for each frame; an example is provided in Fig. 1a;

• Chirp-by-chirp frequency hopping: the starting frequency changes randomly for each chirp; an example is provided in Fig. 1b.

Whereas the former approach is a common solution, also described for example in [6], the latter is here newly proposed.

## A. Frame-by-frame frequency hopping

In frame-by-frame frequency hopping, each radar transmits a frame in a specific $B _ { \mathrm { c h } }$ randomly selected within $B _ { \mathrm { T O T } }$ . All the chirps within the frame are sent using the same $B _ { \mathrm { c h } }$

Assuming a generic attacker, the probability $P _ { \mathrm { f } }$ that the signal from the attacker overlaps in frequency with that of the victim by at least $x _ { \mathrm { f } }$ is

$$
P _ { \mathrm { f } } = \frac { 2 \cdot ( 1 - x _ { \mathrm { f } } ) \cdot B _ { \mathrm { c h } } } { B _ { \mathrm { T O T } } - B _ { \mathrm { c h } } } \cdot \frac { B _ { \mathrm { T O T } } - B _ { \mathrm { c h } } - ( 1 - x _ { \mathrm { f } } ) \cdot B _ { \mathrm { c h } } / 2 } { B _ { \mathrm { T O T } } - B _ { \mathrm { c h } } } \ .\tag{2}
$$

The probability $P _ { \mathrm { { t } } }$ that the signal from the attacker overlaps in time with that of the victim by at least $x _ { \mathrm { t ( F ) } } \in [ 0 , 1 ]$ is

$$
P _ { \mathrm { t ( F ) } } = \operatorname* { m i n } \{ 1 , 2 \cdot \delta \cdot \left( 1 - x _ { \mathrm { t ( F ) } } \right) \} .\tag{3}
$$

The minimum time overlap $x _ { \mathrm { t ( F ) } }$ that causes the loss of the frame can be calculated as

$$
x _ { \mathrm { t ( F ) } } = \left( K _ { \mathrm { c h } } / N _ { \mathrm { c h } } \right) \cdot \left[ \left( T _ { \mathrm { r c h } } / T _ { \mathrm { c h } } \right) \cdot \left( B _ { \mathrm { c h } } / B _ { \mathrm { A D C } } \right) \right] \ .\tag{4}
$$

The first part of (4) considers that at least $K _ { \mathrm { c h } }$ of the $N _ { \mathrm { c h } }$ chirps need to collide to cause a frame collision. The second and third parts of (4) indicate that on average only $T _ { \mathrm { r c h } } / T _ { \mathrm { c h } }$ and $B _ { \mathrm { c h } } / B _ { \mathrm { A D C } }$ of the overlapping chirps actually collide due to variability of initial chirp time or due to the interfering signal being out of ADC bandwidth, respectively.

Given that the two domains are independent, the probability that the attacking signal overlaps with that of the victim in both domains is

$$
\begin{array} { r } { P _ { \mathrm { C f } } = P _ { \mathrm { f } } \cdot P _ { \mathrm { t ( F ) } } \ . } \end{array}\tag{5}
$$

Supposing multiple attackers, acting independently of each other, the probability that at least one of the $N _ { \mathrm { i n t } }$ attackers overlaps with the victim signal in both domains is

$$
P _ { \mathrm { C f N ( F ) } } = 1 - ( 1 - P _ { \mathrm { C f } } ) ^ { N _ { \mathrm { i n t } } } \ .\tag{6}
$$

Assuming as an approximation that each frame can be treated independently of the others, the probability that $M _ { \mathrm { f } }$ consecutive frames collide is

$$
P _ { \mathrm { C M ( F ) } } = P _ { \mathrm { C f N ( F ) } } { } ^ { M _ { \mathrm { f } } } \ .\tag{7}
$$

## B. Chirp-by-chirp frequency hopping

In chirp-by-chirp frequency hopping, each radar transmits each chirp in a different $B _ { \mathrm { c h } }$ , randomly selected within $B _ { \mathrm { T O T } }$

Assuming a generic attacker, the probability $P _ { \mathrm { f } }$ that the generic chirp from the attacker overlaps in frequency with that of the victim by at least $x _ { \mathrm { f } }$ is the same as in (2).

In the time domain, the probability $P _ { \mathrm { { t } } }$ that the generic chirp of the victim overlaps by at least $x _ { \mathrm { t ( C ) } }$ of the chirp, considering that with probability δ the attacker is not transmitting, is

$$
P _ { \mathrm { t ( C ) } } = \delta \cdot \left( 1 - x _ { \mathrm { t ( C ) } } \right) .\tag{8}
$$

Considering that the attacker chirp interferes with the victim chirp only if the signal enters the ADC bandwidth, we can assume that

$$
x _ { \mathrm { t ( C ) } } = 1 - ( T _ { \mathrm { c h } } / T _ { \mathrm { r c h } } ) \cdot ( B _ { \mathrm { A D C } } / B _ { \mathrm { c h } } ) .\tag{9}
$$

Equations 8 and 9 mean that in the extreme case of a very small $B _ { \mathrm { A D C } }$ , the collision occurs only if the chirps exactly overlap, which implies a probability tending to zero; in the opposite extreme, if $B _ { \mathrm { A D C } }$ is as large as $B _ { \mathrm { c h } }$ , then a collision occurs if even a minimum part of the chirps overlaps, which implies a probability tending to $\delta \cdot ( T _ { \mathrm { c h } } / T _ { \mathrm { r c h } } )$

Given that the two domains are independent, the probability that the attacking chirp overlaps with that of the victim in both domains is

$$
P _ { \mathrm { C c h } } = P _ { \mathrm { f } } \cdot P _ { \mathrm { t ( C ) } } .\tag{10}
$$

If multiple attackers are assumed to be acting independently of each other, the probability that at least one of the $N _ { \mathrm { i n t } }$ attackers generates a signal overlapping in both domains to the victim chirp is

$$
P _ { \mathrm { C c h N } } = 1 - ( 1 - P _ { \mathrm { C c h } } ) ^ { N _ { \mathrm { i n t } } } \ .\tag{11}
$$

Assuming that the loss of each chirp is independent of the others, the probability to lose $K _ { \mathrm { c h } }$ or more chirps in the frame of $N _ { \mathrm { c h } }$ chirps is

$$
P _ { \mathrm { C f N ( C ) } } = 1 - \sum _ { i = 0 } ^ { K _ { \mathrm { c h } } - 1 } \binom { N _ { \mathrm { c h } } } { K _ { \mathrm { c h } } } P _ { \mathrm { C c h N } } { } ^ { i } ( 1 - P _ { \mathrm { C c h N } } ) ^ { N _ { \mathrm { c h } } - i } .\tag{12}
$$

Assuming as an approximation that each frame can be treated independently, the probability that $M _ { \mathrm { f } }$ consecutive frames collide is

$$
P _ { \mathrm { C M ( C ) } } = P _ { \mathrm { C f N ( C ) } } { ^ { M _ { \mathrm { f } } } } \ .\tag{13}
$$

## C. Average probability of multiple frame loss

Equations 7 and 13 allow to calculate the probability that $M _ { \mathrm { f } }$ consecutive frames are lost when there are $N _ { \mathrm { i n t } }$ attackers, which we can denote as $P _ { \mathrm { C M ( F ) } } | _ { M _ { \mathrm { f } } }$ and $P _ { \mathrm { C M ( C ) } } | _ { M _ { \mathrm { f } } }$ in the two cases, or more in general as $P _ { \mathrm { C M } } | _ { M _ { \mathrm { f } } }$

Given the results in terms of potential interferers obtained with WiLabVIsim (Section II), the probability of $M _ { \mathrm { f } }$ consecutive frames can be related to the specific scenario removing the parameter $N _ { \mathrm { i n t } }$ . More specifically, the simulations performed in a given scenario output the probability to have no potential interferers $P _ { \mathrm { N i n t } } ( 0 )$ , one potential interferers $P _ { \mathrm { N i n t } } ( 1 )$ , two potential interferers $P _ { \mathrm { N i n t } } ( 2 )$ , and so on.

From the probability vector of a certain number of potential interferers, it is then possible to calculate the average probability that $M _ { \mathrm { f } }$ consecutive frames are lost as

$$
\overline { { P _ { \mathrm { C M } } } } = \sum _ { i = 0 } ^ { \infty } P _ { \mathrm { C M } } | _ { i } \cdot P _ { \mathrm { N i n t } } ( i ) .\tag{14}
$$

This probability can be interpreted as the probability that a given frame is the beginning of an application-level failure.

The results shown in Fig. 2 compare the cases with frame-by-frame and chirp-by-chirp frequency hopping, with two different bandwidth settings. What can be observed is that a limited bandwidth resource and current approaches would yield, in case of increased penetration, a high probability that multiple consecutive frames are lost; for example, looking at Fig. 2a with $B _ { \mathrm { T O T } } { = } 9 0 0$ MHz and 30% penetration, $\overline { { P _ { \mathrm { C M } } } }$ is approximately 0.01, meaning that an event with $M _ { \mathrm { f } } { = } 3$ consecutive lost frames starts on average once every 100 frames. The use of a larger frequency band alone may not be sufficient and the chirp-by-chirp approach cannot help if the bandwidth is limited. Differently, the joint use of a larger frequency band and the chirp-by-chirp frequency hopping appear effective in significantly reducing the impact of interference.

## V. CONCLUSION

In this work, we derive a model to investigate the impact of the interference in a realistic highway scenario when a variable portion of vehicles is equipped with radar sensors based on FMCW. In addition, we proposed a chirp-by-chirp frequency hopping to mitigate the impact of interference and showed that it is effective in improving the system performance when used consistently by all radars, and within a sufficiently large bandwidth allocation. Current deployments within 76-81 GHz may thus face strong limitations due to interference, while new larger allocations in the D-band may provide significant improvement opportunities. The model may be extendable to orthogonal frequency division multiplexing (OFDM) radars, although this aspect will be the subject of our future work.
